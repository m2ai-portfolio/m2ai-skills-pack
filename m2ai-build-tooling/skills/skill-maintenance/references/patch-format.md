# SkillUpgradePatch Format (Phase 2b)

JSON format for automated skill improvement patches generated by audit findings.

## Schema

```json
{
  "skill_name": "example-skill",
  "patch_type": "add_frontmatter",
  "priority": 1,
  "description": "Add missing frontmatter block with name and description",
  "suggested_change": "Add ---\\nname: example-skill\\ndescription: ...\\n--- to top of SKILL.md",
  "audit_score_before": 35,
  "audit_criterion": "FM-01",
  "auto_applicable": true
}
```

## Fields

| Field | Type | Description |
|-------|------|-------------|
| skill_name | string | Name of the skill being patched |
| patch_type | enum | Type of patch (see Patch Types below) |
| priority | 1-3 | 1=quick fix, 2=moderate effort, 3=major rework |
| description | string | Human-readable description of the change |
| suggested_change | string | Specific change to make (may be a diff, instruction, or template) |
| audit_score_before | number | Skill's score before the patch |
| audit_criterion | string | ID of the audit criterion that triggered this patch (e.g., FM-01, NM-01) |
| auto_applicable | boolean | Whether this patch can be applied without human review |

## Patch Types

| Type | Priority | Auto-applicable | Description |
|------|----------|-----------------|-------------|
| add_frontmatter | 1 | YES | Add missing `---` frontmatter block |
| fix_name | 1 | YES | Fix name to match directory or fix format |
| fix_name_format | 1 | YES | Convert name to lowercase-hyphens |
| add_trigger_phrases | 2 | no | Add "when" context to description |
| split_progressive | 2 | no | Move content from SKILL.md to references/ |
| fix_script_refs | 2 | no | Change Read tool refs to Bash execution |
| add_allowed_tools | 3 | no | Add allowed-tools field to frontmatter |
| add_model_field | 3 | no | Add model field to frontmatter |
| restructure_dirs | 3 | no | Add scripts/references/assets directories |

## Auto-Applicable Rules

A patch is `auto_applicable: true` ONLY when:
1. The change is mechanical (no judgment required)
2. The change cannot break skill functionality
3. The change is fully reversible

Currently only three patch types qualify:
- `add_frontmatter` — template frontmatter from directory name
- `fix_name` — set name to match directory
- `fix_name_format` — lowercase + hyphens conversion

All other patches require HIL (human-in-the-loop) review before application.

## Integration with Skill Registry

Patch fields map to the skill-registry-schema.json learning block:
- `patch_type` -> `learning.improvement_type`
- `audit_criterion` -> `learning.source_criterion`
- `audit_score_before` -> `learning.baseline_score`
- `auto_applicable` -> `learning.requires_review` (inverted)
